五、SQL Injection

1-1. Low

  • source

1-2. 漏洞分析

  • 对 id 没有任何过滤，直接在数据库中查询‘id’的值，存在 sql 注入

2-1. Mid

  • source

2-2. 注解

  • group_concat(column_name) from information_schema.columns where table_name=’users’：
    • group_concat():表示将多个字段合成一个字段，多个 group by 配合使用
    • column_name:列名;table_name:表名
    • information_schema.columns:是 mysql 中的元数据表
    • table_schema:表所在的数据库名
      2-3. 漏洞分析
  • 此处只是对参数 id 进行了转义，
    2-3. 漏洞分析
  • 此处经过判断存在数字型注入，所以 mysql_real_escape_string()函数不能使用
  • 继续按步骤手工注入即可

3-1. High

  • source

2-2. 漏洞分析

  • 这是只是添加了 limit 1，以此限制输入，但是可以通过# 来注释掉
  1. 预防

4-2. 防范分析

  • 采用了 PDO 技术，划清了代码与数据的界限，有效防御 SQL 注入，同时只有返回的查询结果数量为一时，才会成功输出，这样就有效预防了“脱裤”，Anti-CSRFtoken 机制的加入了进一步提高了安全性。

4-3. sql 手工

  1. 判断是否存在注入，注入是字符型还是数字型

  2. 猜解 SQL 查询语句中的字段数

  3. 确定显示的字段顺序

  4. 获取当前数据库

  5. 获取数据库中的表

  6. 获取表中的字段名

  7. 下载数据
