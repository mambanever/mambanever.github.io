二、Command injection

1.1 Low

  • source：

1.2 注解:

  • $_REQUEST[]:$_REQUEST 变量可用来收集通过 GET 和 POST 方法发送的表单数据

  • $_GET[]:预定义的 $_GET 变量用于收集来自 method=”get” 的表单中的值

  • $_POST[]:预定义的 $_POST 变量用于收集来自 method=”post” 的表单中的值

  • stristr(string,search):搜索 string 字符串在 search 中第一次出现

  • php_uname(date):返回运行 PHP 的系统的有关信息。此处的 s 表示操作系统

  • shell_exec()和 exec():都可以运行 Linux 命令，shell_exec()返回全部数据，exec()返回一行数据
    1.3 漏洞利用

  • window 和 linux 系统都可以用&&来执行多条命令，
    127.0.0.1&&net user
    同样的命令注入符号还有：“|“ 、”||“、”&“等

    2.1 Mid

  • source:

2.2 注解:

  • array_keys()函数:返回包含数组中所有键名的一个新数组。

  • str_replace(find,replace)函数:替换字符串中的一些字符(区分大小写)

  • “&&“与”&“的区别是：前者是符号前面的命令执行成功后才可执行符号后面的命令，否者不执行后面的命令，后者是符号前面的命令是否执行成功都会执行符号后面的命令。

    2.3 漏洞利用

  • 绕过即可：127.0.0.1&;&net user 或 127.0.0.1||net user

    3.1 High

  • source:

3.2 注解:

  • explode(separator,string):使用一个字符串分割另一个字符串，并返回由字符串组成的数组
    3.3 漏洞分析
  • 黑名单看似过滤了所有的非法字符，但仔细观察到是把”| ”（注意这里|后有一个空格）替换为空字符，于是 ”|”成了“漏网之鱼”
  1. 预防
  • source

4.1 漏洞分析

  • Impossible 级别的代码加入了 Anti-CSRF token，同时对参数 ip 进行了严格的限制，只有诸如“数字.数字.数字.数字”的输入才会被接收执行，因此不存在命令注入漏洞。

总结

  • Low：无任何过滤，知道注入的形式
  • Mid：对于关键字只是简单的替换，而且还是空字符串替换
  • High：原题是更多地黑名单，但是现在变成—字符串的过滤+字符串的分割+数字判断+长度判断。
