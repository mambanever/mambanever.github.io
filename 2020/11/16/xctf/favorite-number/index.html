题目：千方百计的绕过，很直接。

知识点：

  1. 数组强等于溢出绕过。
  2. 跨行检测(%0a)绕过数字检测
  3. 黑名单绕过，如 tac 查看 flag，
  4. inode 索引结点寻找 flag 文件

0x01

首先看到

可以确定应该是到代码审计的题目，dirserch 扫描后台，index.php 也没有任何变化，
检查代码:

  1. 和数组强相等，第一个元素不能等于 admin
  2. 正则表达式，要求 num 全是数字。
  3. 黑名单排除。
  4. 注释 PHP5.5.9，一个提示，可能需要运行 PHP 的漏洞

可是数组如何强相等？没有思路。

wp 提示：PHP 安全

所以尝试了下构造的 payload

stuff[4294967296]=admin&stuff[1]=user&num=123
可以访问，说明成功绕过第一条件。

wp 提示：可以用跨行检测来绕过数字检测，用换行符%0a

用 hackbar 访问没回显，bp 抓包，发现重新编码 ，导致 hackbar 没反应
bp 尝试，成功访问，绕过了数字检测。
最后需要绕过黑名单，命令执行访问文件。

wp 提示：访问文件的方法，以及可以通过 iNode 索引节点来找到 flag
构造 payload，bp 访问，得到 flag 的序号

tac 方法查看，得到 flag。
