web攻击方法理解

2020-10-15 17:19:50
  • web安全
  • web

一、Brute force

1、why

由于用户有可能设置弱口令或其他容易的口令，所以可以通过密码字典来穷举用户可能的所有密码破解，简单粗暴。

2、how

  • 准备质量比较高的密码字典。
  • 正确是用工具进行‘撞库’。如bp使用。

3、defense

设置更加复杂的密码。

二、Command injection（exp:ping ip)

1.why

命令注入漏洞产生的原因一般是：web应用程序有时需要调用一些系统命令的函数,如PHP中的system,exec,shell-exec等等,当用户能够控制这些函数中的参数时,就可以将恶意参系统命令拼接到正常命令中,从而造成命令注入攻击并且，用户的输入未经过滤或者过滤不严格就直接被当做系统命令进行执行；因此，嘿壳儿就可以通过批处理中的一些技巧来一次执行多条命令，并且通过构造可以这行任意的命令，从而控制我们的计算机或服务器。

漏洞形成需要同时满足以下三个条件:
1.使用了内部调用shell的函数(system,exec)
2.将外界传入的参数传递给内部调用shell的函数
3.参数中shell的元字符没有被转义

2.how

通过构造服务端能够识别的恶意语句（shell命令）来达到目的。

在命令执行的过程中，常用的连接符有：&& 、 & 、 || 、 | 、；
**&&** : 前一个指令执行成功，后面的指令才继续执行，就像进行与操作一样
**&** ： 直接连接多个命令
**||** ： 前 一个命令执行失败，后面的才继续执行，类似于或操作
**|** ： 管道符，将前一个命令的输出作为下一个命令的输入
**;** ： 直接连接多个命令。

3、defense

可以设置白名单，限制用户输入。

三、CSRF(跨站请求伪造)

1.why

这个例子很形象，反正我是懂了。

受害者 Bob 在银行有一笔存款，通过对银行的网站发送请求 http://bank.example/withdraw?account=bob&amount=1000000&for=bob2 可以使 Bob 把 1000000 的存款转到 bob2 的账号下。通常情况下，该请求发送到网站后，服务器会先验证该请求是否来自一个合法的 session，并且该 session 的用户 Bob 已经成功登陆。

​ 黑客 Mallory 自己在该银行也有账户，他知道上文中的 URL 可以把钱进行转帐操作。Mallory 可以自己发送一个请求给银行：http://bank.example/withdraw?account=bob&amount=1000000&for=Mallory。但是这个请求来自 Mallory 而非 Bob，他不能通过安全认证，因此该请求不会起作用。

​ 这时，Mallory 想到使用 CSRF 的攻击方式，他先自己做一个网站，在网站中放入如下代码： src=”http://bank.example/withdraw?account=bob&amount=1000000&for=Mallory ”，并且通过广告等诱使 Bob 来访问他的网站。当 Bob 访问该网站时，上述 url 就会从 Bob 的浏览器发向银行，而这个请求会附带 Bob 浏览器中的 cookie 一起发向银行服务器。大多数情况下，该请求会失败，因为他要求 Bob 的认证信息。但是，如果 Bob 当时恰巧刚访问他的银行后不久，他的浏览器与银行网站之间的 session 尚未过期，浏览器的 cookie 之中含有 Bob 的认证信息。这时，悲剧发生了，这个 url 请求就会得到响应，钱将从 Bob 的账号转移到 Mallory 的账号，而 Bob 当时毫不知情。等以后 Bob 发现账户钱少了，即使他去银行查询日志，他也只能发现确实有一个来自于他本人的合法请求转移了资金，没有任何被攻击的痕迹。而 Mallory 则可以拿到钱后逍遥法外。

也就是，用户A访问信任网站C，成功登入后，留下了cookie，然后访问了攻击者B设置恶意代码，在用户A不知情的情况下，访问了网站C，并且执行了恶意代码的命令，达成目的。

2、defense

  1. 验证 HTTP Referer 字段。

  2. 在请求地址中添加 token 并验证

  3. 在 HTTP 头中自定义属性并验证

  4. 验证码

五、SQL Injection

1、why

通过sql命令构造恶意语句。

2、how

  • 手工注入(注意使用#或–+来注释掉后面的内容)

    1.判断是否存在注入，注入是字符型还是数字型(数字型没有引号)

    ​ 1,1’,1’ and 1=1,1’ and 1=2

    2.猜解SQL查询语句中的字段数

    ​ order by 1，或 1’ or order by 1 #

    3.确定显示的字段顺序

    ​ order by 来数字尝试，或者union select 1,2~~

    4.获取当前数据库

    ​ 已经找到了注入点（此处为2），然后用sql语句替换注入点（选择 用database()替换2）。

    5.获取数据库中的表

    ​ 继续用sql语句替换

    6.获取表中的字段名

    7.下载数据

  • 使用工具注入，sqlmap工具。

3、defense

可以预备义处理和参数式查询。

六、SQL Injection(Blind)

1、why

手工盲注的过程，就像你与一个机器人聊天，这个机器人知道的很多，但只会回答“是”或者“不是”，因此你需要询问它这样的问题，例如“数据库名字的第一个字母是不是a啊？”，通过这种机械的询问，最终获得你想要的数据。

盲注分为基于布尔的盲注、基于时间的盲注以及基于报错的盲注，这里由于实验环境的限制，只演示基于布尔的盲注与基于时间的盲注。

2、how

下面简要介绍手工盲注的步骤（可与之前的手工注入作比较）：

1.判断是否存在注入，注入是字符型还是数字型

2.猜解当前数据库名

先猜库名的长度，然后遍历库名，真的烦。。。。

3.猜解数据库中的表名

猜，猜，猜。。。

4.猜解表中的字段名
