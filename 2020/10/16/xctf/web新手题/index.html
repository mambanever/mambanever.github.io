1.view_source

直接 F12,查看源代码，找到 flag。

2.robots

题目知识点:robots 协议也叫 robots.txt（统一小写）是一种存放于网站根目录下的 ASCII 编码的文本文件，它通常告诉网络搜索引擎的漫游器（又称网络蜘蛛），此网站中的哪些内容是不应被搜索引擎的漫游器获取的，哪些是可以被漫游器获取的。因为一些系统中的 URL 是大小写敏感的，所以 robots.txt 的文件名应统一为小写。（百度）
解法：选择直接在 URL 上寻找 robots.txt 文件，得到下图：

URL 再次访问 flag 文件，得到 flag。

3.backup

知识点：index.php 的备份文件有“.bak”后缀，所以 URL 直接寻找备份文件，下载的文件更改后缀名为.php,打开文件，得到 flag。

4.cookie

知识点：Cookie，有时也用其复数形式 Cookies。类型为“小型文本文件”，是某些网站为了辨别用户身份，进行 Session 跟踪而储存在用户本地终端上的数据（通常经过加密），由用户客户端计算机暂时或永久保存的信息（百度）也就是存储的浏览记录。
方法一：F12 查看源代码，在 network 模块的 response 下找到 look-here=cookie.php，所以选择在 URL 上选择，然后跳转页面如下：

继续 F12，找到 flag。
方法二：针对 cookie，可以选择用 burpsuite 抓包，找到 cookie.php,得 flag。

5.disabled_button

知识点：HTML 语法。
解法：直接 F12,去掉 button 元素中 disable 属性，点击按键，得到 flag。

6.weak_auth

简单密码认证，直接通过 burpsuite 简单密码字典爆破，在 BP 上 123456 的长度不一样，得到密码，得 flag。

7.command_execution（命令执行）

知识点：
ping 功能
命令执行，有很多种方式，常用的命令执行：

WAF 介绍
Linux 常用的命令
解法：一、用”ls”,”../“,”cat”等操作命令，一步一步寻找到 flag.txt 文件
二、可以直接使用：find / -name flag.txt 找到文件，然后用 cat 打开文件，得到 flag。

8.simple_php

知识点：简单的代码审计问题，需要基本能够看懂 PHP 代码的意思.
解法：用 get 方式传送，直接 URL 上操作，通过 PHP，得 a>0&&b 不是数字，且要大于 1234.得 flag。

“?”此处是分隔符号。

9.get_post

知识点：HTTP 中两种请求，get 和 post。
解法：get 方式，直接用 URL，post 在源代码 network 模块中传送。
得到 flag。

10.xff_referer

知识点:
X-forwarded-For 是告诉服务器当前请求者的最终 ip 的 http 请求头字段,通常可以直接通过修改 http 头中的 X-Forwarded-For 字段来仿造请求的最终 ip。
referer 是 HTTP 表头的一个字段，就是告诉服务器当前访问者是从哪个 url 地址跳转到自己的，跟 xff 一样，referer 也可直接修改。
解法：直接选择代理，使用 burp suite 抓包，然后 to request 模块，修改 xff 和 referer，得到 flag。

11.webshell

知识点：一句话木马,中国菜刀/蚁剑.
解法：直接菜刀，解除 flag。

12.simple_js

知识点：js 在 HTML 中的使用，能够看懂 web 上的源代码。同时结合了密码学。
解法：在源代码中找到了一些奇怪的数字，

十六进制转换成十进制，然后找到其对应的 ascii 码，得到一些数字，然后在这些数通过 ascii 转换成字母，注意得到的字符串不是 flag，拼凑正确格式，得到 flag。
